Basic Examples

Simple handlers to get you started with Contextualizer

URL Detector

Regex Handler

Detects URLs in clipboard and shows a notification

{
  "name": "URL Detector",
  "description": "Detects URLs and shows notification",
  "type": "regex",
  "regex": "https?://[\\w\\.-]+(?:/[\\w\\.-]*)*",
  "auto_focus_tab": false,
  "bring_window_to_front": false,
  "actions": [
    {
      "name": "show_notification",
      "message": "URL detected: {_matched_content}"
    }
  ]
}
Usage: Copy any URL to clipboard to see the notification

Email Formatter

Regex Handler

Formats email addresses and copies them back to clipboard

{
  "name": "Email Formatter",
  "description": "Formats email addresses",
  "type": "regex",
  "regex": "([\\w\\.-]+)@([\\w\\.-]+)",
  "auto_focus_tab": false,
  "actions": [
    {
      "name": "copy_to_clipboard",
      "key": "_formatted_email"
    },
    {
      "name": "show_notification",
      "message": "Email formatted: {_formatted_email}"
    }
  ],
  "context_processors": [
    {
      "key": "_formatted_email",
      "value": "üìß {$1}@{$2}"
    }
  ]
}
Usage: Copy an email address to get it formatted with an emoji

UI Best Practices

Recommended

  • Use auto_focus_tab: false for background processing
  • Use bring_window_to_front: false as default
  • Reserve window activation for high-priority content
  • Use notifications for simple confirmations

Avoid

  • Multiple handlers with bring_window_to_front: true
  • Auto-focusing tabs for frequent operations
  • Interrupting user workflow unnecessarily
  • Opening windows for every clipboard change

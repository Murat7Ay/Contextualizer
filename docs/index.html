Get Started

Installation

Get Contextualizer up and running on your system

Portable Single EXE (Network Share)

Copy the single EXE from the network share and run it without installation

# Create target folder (if needed)
New-Item -Path 'C:\PortableApps\Contextualizer' -ItemType Directory -Force

# Copy the EXE from the network share
Copy-Item "\\ortak\cashmanagement\murat ay\contextualizer\*.exe" "C:\PortableApps\Contextualizer\" -Force

# Run
# Go to C:\PortableApps\Contextualizer and launch the EXE

Build from Source (GitHub)

Clone from GitHub, build locally, and run

# Clone the repository
git clone https://github.com/Murat7Ay/Contextualizer.git
cd Contextualizer

# Build the project
dotnet build --configuration Release

# Run the application
dotnet run --project WpfInteractionApp

Portable Directory Structure

  • C:\PortableApps\Contextualizer\Config ‚Äî Settings and handlers.json (auto-created on first run)
  • C:\PortableApps\Contextualizer\Data\Exchange ‚Äî Exchange templates (seeds sample-regex-handler.json if empty)
  • C:\PortableApps\Contextualizer\Data\Installed ‚Äî Installed handlers
  • C:\PortableApps\Contextualizer\Data\Logs ‚Äî Application logs
  • C:\PortableApps\Contextualizer\Plugins ‚Äî Plugins
  • C:\PortableApps\Contextualizer\Temp ‚Äî Temporary files

These are created by CreatePortableDirectories() on first launch. A sample handler is generated by CreateSampleExchangeHandler().

System Requirements

  • OS: Windows 10/11 (x64)
  • Framework: .NET 9.0 or later
  • Memory: 256 MB RAM minimum
  • Storage: 100 MB available space

Quick Start

Get up and running with Contextualizer in 5 minutes

1

Launch the Application

Start WpfInteractionApp.exe. The application will begin monitoring your clipboard automatically.

2

Get Your First Handler from Exchange

On first launch, a sample is seeded at C:\PortableApps\Contextualizer\Data\Exchange\sample-regex-handler.json. You can also drop this JSON into the Exchange folder:

{
  "name": "Hello",
  "title": "First handler",
  "type": "regex",
  "screen_id": "markdown2",
  "bring_window_to_front": true,
  "auto_focus_tab": true,
  "regex": "\\b\\w{3,}\\b",
  "actions": [
    {
      "name": "show_window",
      "key": "_formatted_output"
    }
  ],
  "output_format": "üëã Hello $(_match)
Welcome!!"
}
3

Test the Hello Handler

Copy any word with 3+ characters. Contextualizer will match it and open a window (screen markdown2) showing the formatted message.

4

Install More from Exchange

Place handler templates (*.json) into C:\PortableApps\Contextualizer\Data\Exchange. They will be discovered on startup.

5

Explore Advanced Features

Check out the examples section for more complex handlers and automation scenarios.

Handler Configuration Reference

The unified handler configuration model supports all handler types via a single JSON schema. Properties below map to HandlerConfig and are used selectively depending on handler type.

Core Properties

Property Type Description
namestringUnique name for the handler
descriptionstringOptional description for documentation
typestringHandler type (e.g., regex, api, database, file, lookup, synthetic, cron, manual)
titlestringUI title when showing windows/tabs
screen_idstringTarget screen/view identifier (e.g., markdown, markdown2, url_viewer)
validatorstringCustom validator key (optional)
context_providerstringCustom context provider key (optional)
actionsarrayList of action objects to execute (see Actions section)
output_formatstringTemplate for formatted output (supports $(_match) and keys)
user_inputsarrayInteractive input requests required before execution
requires_confirmationboolPrompt for confirmation before executing actions

UI Behavior

PropertyTypeDescription
auto_focus_tabbool (default: false)Focus the opened tab immediately
bring_window_to_frontbool (default: false)Force the main window to the foreground

Regex / File / Lookup

PropertyTypeDescription
regexstringPattern to match clipboard/content
groupsarray<string>Explicit group names (optional)
pathstringFile path or directory (File/Lookup handlers)
delimiterstringDelimiter for parsing text files
key_namesarray<string>Column/field names for keys
value_namesarray<string>Column/field names for values
file_extensionsarray<string>Allowed file extensions

API Handler

PropertyTypeDescription
urlstringEndpoint URL
methodstringHTTP method (GET, POST, etc.)
headersobjectRequest headers
request_bodyobjectJSON request payload
content_typestringRequest content type
timeout_secondsnumberRequest timeout

Database Handler

PropertyTypeDescription
connectionStringstringDatabase connection string
querystringQuery or command to execute
connectorstringDB provider/connector identifier
command_timeout_secondsnumberCommand timeout
connection_timeout_secondsnumberConnection timeout
max_pool_sizenumberMaximum connection pool size
min_pool_sizenumberMinimum connection pool size
disable_poolingboolDisable connection pooling

Synthetic Handler

PropertyTypeDescription
reference_handlerstringName of existing handler to reference
actual_typestringReal handler type executed behind the scenes
synthetic_inputobjectPredefined input to feed the referenced handler

Cron Handler

PropertyTypeDescription
cron_job_idstringUnique job id
cron_expressionstringCron schedule expression
cron_timezonestringTime zone id for scheduling
cron_enabledbool (default: true)Enable/disable the job

Application Settings

Application settings are persisted under C:\PortableApps\Contextualizer\Config\appsettings.json and map to AppSettings.

Core Paths

  • handlers_file_path: Path to handlers.json
  • plugins_directory: Folder for plugins
  • exchange_directory: Folder for Exchange templates

Keyboard Shortcut

  • keyboard_shortcut.modifier_keys: Array of modifiers (e.g., ["Ctrl"])
  • keyboard_shortcut.key: Main key (e.g., W)

Timing

  • clipboard_wait_timeout (ms)
  • window_activation_delay (ms)
  • clipboard_clear_delay (ms)

Window Settings

  • window_settings: width, height, left, top, window_state, grid_splitter_position
  • window_settings.settings_window/exchange_window: left, top, width, height

UI Settings

  • ui_settings.toast_position_x, toast_position_y
  • ui_settings.theme (e.g., Dark)
  • ui_settings.skipped_update_version, last_update_check
  • ui_settings.network_update_settings: enable_network_updates, network_update_path, check_interval_hours, auto_install_non_mandatory, auto_install_mandatory

Logging Settings

  • logging_settings.enable_local_logging, enable_usage_tracking
  • logging_settings.local_log_path, usage_endpoint_url
  • logging_settings.minimum_log_level, max_log_file_size_mb, max_log_file_count, enable_debug_mode

Config System

  • config_system.enabled
  • config_system.config_file_path, secrets_file_path
  • config_system.auto_create_files, file_format

Settings UI

You can manage these settings from the application UI:

  • Application Settings window (SettingsWindow): Paths, keyboard shortcut, timing, config system, network updates
  • Logging Settings window (LoggingSettingsWindow): Local logging, analytics, and log statistics

UI Controls

Master the Contextualizer interface and user experience features

Tab Management

Chrome-like Tab Behavior

  • Middle-click to close: Click mouse wheel on any tab to close it
  • Close button: Click the X button on each tab
  • Silent tabs: New tabs open in background by default
  • Smart focus: Configurable tab focusing behavior

Window Management

Intelligent Window Behavior

  • Smart activation: Only brings window to front when needed
  • Minimize restore: Automatically restores minimized windows
  • Configurable: Control window behavior per handler

Activity Logging

Real-time Activity Feed

  • Activity log: See all handler executions in real-time
  • Log filtering: Filter by log level and search text
  • Search logs: Quickly find specific log entries
  • Clear logs: Clean up the log view when needed

      "key": "_formatted_output"

Configuration Options

  • auto_focus_tab: false - Tab opens in background (recommended)
  • auto_focus_tab: true - Tab becomes active immediately
  • bring_window_to_front: false - Window stays in current state (recommended)
  • bring_window_to_front: true - Window comes to front (use sparingly)

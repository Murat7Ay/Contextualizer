Dispatch Execution Flow (Detailed)

1
CanHandle
Each handler checks if it can process the captured content (regex match, type checks, etc.). Non-matching handlers exit early.
2
Confirmation (optional)
If requires_confirmation = true, the user is prompted. Cancel stops execution.
3
User Inputs (wizard)
Collect inputs with Next/Back/Cancel; supports dependent selection lists and validation. Merged into context by input key.
4
Seed Context
constant_seeder merged first (raw). Then seeder is resolved and merged.
5
Dynamic Resolution (multi-pass)
  • $file:path â†’ read file content
  • $config:section.key â†’ load from config/secret files
  • $func: â†’ evaluate function pipeline (regular + pipe)
  • $(key) â†’ final placeholder replacement from context
6
Ensure Output
Set _self (context JSON) if missing. Render _formatted_output from output_format if provided; else fallback to _self.
7
Action Loop
  • Action Confirmation: if action requires_confirmation = true, prompt user; cancel skips this action.
  • Action User Inputs: if action has user_inputs, run the same wizard flow (Next/Back/Cancel); merge results into an action-local context.
  • Action Conditions: evaluate conditions (and/or, equals/contains/regexâ€¦) against the (possibly updated) action context; skip if false.
  • Action-level Seeding: merge action constant_seeder, then resolve and merge action seeder using the same multi-pass order; base handler context is not mutated.
  • Key Usage: many plugins use key to read a value from context (e.g., show_window.key â†’ content to render).
  • Execute: run plugin (e.g., show_notification, show_window, copy_to_clipboard, open_file), respecting auto_focus_tab and bring_window_to_front.
  • inner_actions: execute nested actions after parent using the action-local context.
{
  "name": "show_window",
  "requires_confirmation": true,
  "user_inputs": [
    { "key": "title_override", "title": "Title", "message": "Enter a title", "is_required": false }
  ],
  "constant_seeder": { "screen_id": "markdown" },
  "seeder": { "_formatted_output": "$func:string.upper($(body))" },
  "conditions": { "operator": "and", "conditions": [ { "operator": "contains", "field": "_self", "value": "$(needle)" } ] },
  "key": "_formatted_output",
  "inner_actions": [
    { "name": "copy_to_clipboard", "key": "_formatted_output" }
  ]
}
8
Parallelism & Logging
Handlers run in parallel; execution and errors are logged. User sees real-time feedback via activity log and toasts.

Dynamic Value Resolution

$(key)
Replaced with the value from current context. Example: $(customer_id) â†’ 12345.
$config:section.key
Reads from config file defined in settings (ConfigSystem.ConfigFilePath/SecretsFilePath). Example: $config:api_keys.jira_token.
$func:
Evaluated by the function pipeline (regular and $func:{{ ... | ... }}). Supports url.*, web.*, ip.*, json.*, string.*, math.*, array.* and base functions.
$file:path
Loads file content and then proceeds with function/placeholder resolution on the loaded text.

Dispatch Details

Resolution Order
$file: include â†’ $config: lookup â†’ $func: evaluation â†’ $(key) placeholders.
constant_seeder vs seeder
constant_seeder merges raw constants first; seeder is resolved (functions/placeholders) and then merged.
Action Conditions
Each action can have conditions. They are evaluated (and/or, equals, contains, regex, etc.) against the current context; failing actions are skipped.
inner_actions
Actions can contain inner_actions. They execute after the parent action with the same context (including newly seeded values).
Output Formatting
If output_format is set, _formatted_output is rendered from context; otherwise falls back to _self (context JSON).

Function Processor

Supports regular ($func:name(args)) and pipeline ($func:{{ value | fn(args) | fn2() }}) functions with chaining.

Base
today, now, yesterday, tomorrow, guid, random, base64encode, base64decode, env, username, computername
url.*
encode, decode, domain, path, query, combine
web.*
get, post, put, delete
ip.*
local, public, isPrivate, isPublic
json.*
get, length, first, last, create
string.*
upper, lower, trim, replace, substring, contains, startsWith, endsWith, split, length
math.*
add, subtract, multiply, divide, round, floor, ceil, min, max, abs
array.*
get, length, join

Order with context: functions are processed before final $(key) replacement; placeholders inside parameters are resolved. Supports $file: include and $config: lookups.

Condition Evaluator

Logical
and, or with nested conditions
Comparisons
equals, not_equals, greater_than, less_than
Text
contains, starts_with, ends_with, matches_regex
Null/Empty
is_empty, is_not_empty

Values can reference context via $(key), resolved before comparison.

Handler Type Details

API Handler

Matches optional regex, builds request from url, method, headers, and optional request_body. Flattens JSON responses into context.

BehaviorDetails
RegexOptional, compiled with 5s timeout; named groups preferred; adds _match and group keys
Requesturl after dynamic resolution; method default GET; headers merged; request_body resolved via placeholders/functions
ResponseAdds StatusCode, IsSuccessful, RawResponse; JSON responses flattened to keys (e.g. data.items[0].id)
TimeoutsClient timeout from timeout_seconds (default 30s); connection pooling optimized

Database Handler

Executes read-only SQL with parameterized inputs; builds a tabular context and renders markdown by default.

BehaviorDetails
SafetyOnly SELECT allowed; forbidden keywords blocked; parameters truncated to 4000 chars
Regex paramsOptional regex; sets p_input, p_match, and groups (group names or p_group_i)
Connectorsconnector alias: mssql @, plsql :; connection via ConnectionManager
OutputContext contains _count and keys like Column#Row; output_format or auto markdown table

File Handler

Processes file clipboard content; validates extensions; exposes rich file metadata for each file.

KeysExample
BasicsFileName0, FullPath0, Extension0, SizeBytes0
TimestampsCreationDate0, LastAccess0, LastWrite0 (and UTC variants)
AttributesHidden0, ReadOnly0, System0, Archive0, ...
Counts_count = number of files

Lookup Handler

Loads key-value data from a delimited file; resolves path via config, supports comments and {{NEWLINE}} replacement.

BehaviorDetails
LoadResolves path with config; reads file; splits by delimiter
Schemakey_names choose dictionary keys; value_names create value fields
MatchCanHandle true if clipboard text matches a loaded key
